When the call to a function doesn't have any side effects, what is the point of making the call if the results are ignored? In such case, either the function call is useless and should be dropped or the source code doesn't behave as expected.

This rule raises an issue when a builtin function or methods which has no side effects is called and its result is not used.

Noncompliant Code Example

list([1, 2])  # Noncompliant

Compliant Solution

myvar = list([1, 2])

Exceptions

No issue will be raised when the function or method call is the only statement in a try...except body. This usually indicates that an exception is expected, and this exception is the side-effect.

def tryExcept():
    d = {}
    try:
        d[1]
    except IndexError as e:
        pass

    try:
        int("abc")
    except ValueError as e:
        pass
